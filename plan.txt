## A. PROJECT SUMMARY

### App Overview

**BuildACompleteInventory** is a multi-tenant inventory management platform for supermarkets, grocery stores, retail chains, mini marts, wholesale distributors, and warehouses. It centralizes product catalog management, real-time stock tracking across stores and warehouses, procurement (suppliers → purchase orders → goods receipt), inter-warehouse transfers, sales and returns, batch/expiry tracking, low-stock monitoring, inventory valuation, operational reporting, role-based access, and full audit history.

### User Roles (via `UserRole` + `AppUserRole` OptionSet — not separate UserTypes)

| Role | Primary Responsibilities |
|------|--------------------------|
| OrganizationAdmin | Tenant setup, users, stores, warehouses, global settings |
| StoreManager | Store operations, sales, returns, local stock view |
| WarehouseManager | Warehouse stock, transfers, receipts, adjustments |
| PurchaseManager | Suppliers, POs, goods receipt |
| InventoryClerk | Stock counts, adjustments, batch management |
| SalesStaff | Sales orders, returns (POS-lite) |
| Accountant | Reports, valuation (read + export) |
| Viewer | Read-only across assigned scope |

### Modules (MVP)

1. **Tenant & Org Setup** — Organization, stores, warehouses, users, invitations  
2. **Product Catalog** — Categories, products, UOM, barcodes, pricing  
3. **Inventory Core** — Warehouse stock, batches, movements, adjustments  
4. **Procurement** — Suppliers, purchase orders, goods receipt  
5. **Stock Transfers** — Inter-warehouse transfers with approval flow  
6. **Sales & Returns** — Sales orders, sales returns, stock deduction/restoration  
7. **Alerts & Notifications** — Low-stock, expiry-soon, in-app + email  
8. **Dashboard & Reports** — KPIs, stock valuation, movement history  
9. **Audit & Security** — AuditLog, login audit, role-scoped data access  

### MVP vs Later Phases

| MVP (Phase 1 — this blueprint) | Phase 2 | Phase 3 |
|--------------------------------|---------|---------|
| Multi-tenant org, stores, warehouses | Barcode scanner integration | EDI / supplier API sync |
| Product catalog + batches/expiry | POS hardware integration | Demand forecasting / AI reorder |
| PO → Goods Receipt → Stock | Recurring PO templates | Multi-currency / tax engines |
| Stock transfers + adjustments | Customer master + CRM-lite | Marketplace integrations |
| Sales orders + returns | Advanced pricing (tiers, promos) | Mobile offline mode (Flutter) |
| Low-stock + expiry alerts | Scheduled report email delivery | Warehouse bin/slot management |
| Dashboard + tabular reports | OAuth social login | IoT scale / RFID hooks |
| OTP + password auth, RBAC | Push notifications (mobile) | Franchise / subsidiary hierarchies |

## Appendix: Key Business Flows

| Flow | Path |
|------|------|
| **Purchase-to-Stock** | Supplier → PO (Draft→Approved) → GoodsReceipt Confirm → `WarehouseStock +=` → `StockBatch` (if tracked) → `InventoryMovement (PurchaseReceipt, In)` → PO line received qty updated |
| **Sale-to-Deduct** | SalesOrder Draft → `StockAvailabilityCheck` → Confirm → `WarehouseStock -=` → FIFO batch deduct → `InventoryMovement (SalesIssue, Out)` |
| **Transfer** | StockTransfer Draft→Approved→Completed → source reserved then deducted, dest increased → TransferOut + TransferIn movements |
| **Low-Stock Alert** | `WarehouseStock.OnUpdate` or daily `LowStockAlertJob` → `StockAlert` → InApp + Email via `LOW_STOCK_ALERT` template |
